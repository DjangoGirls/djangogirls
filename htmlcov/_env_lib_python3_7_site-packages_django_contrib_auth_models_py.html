33class Permission(models.Model): 

34 """ 

35 The permissions system provides a way to assign permissions to specific 

36 users and groups of users. 

37 

38 The permission system is used by the Django admin site, but may also be 

39 useful in your own code. The Django admin site uses permissions as follows: 

40 

41 - The "add" permission limits the user's ability to view the "add" form 

42 and add an object. 

43 - The "change" permission limits a user's ability to view the change 

44 list, view the "change" form and change an object. 

45 - The "delete" permission limits the ability to delete an object. 

46 

47 Permissions are set globally per type of object, not per specific object 

48 instance. It is possible to say "Mary may change news stories," but it's 

49 not currently possible to say "Mary may change news stories, but only the 

50 ones she created herself" or "Mary may only change news stories that have a 

51 certain status or publication date." 

52 

53 Three basic permissions -- add, change and delete -- are automatically 

54 created for each Django model. 

55 """

258 True. Thus, a user who has permission from a single auth backend is 

259 assumed to have permission in general. If an object is provided, check 

260 permissions for that object. 

261 """ 

262 # Active superusers have all permissions. 

263 if self.is_active and self.is_superuser:
