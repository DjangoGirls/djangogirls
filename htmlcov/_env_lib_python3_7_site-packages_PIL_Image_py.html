862 def convert(self, mode=None, matrix=None, dither=None, palette=WEB, colors=256): 

863 """ 

864 Returns a converted copy of this image. For the "P" mode, this 

865 method translates pixels through the palette. If mode is 

866 omitted, a mode is chosen so that all information in the image 

867 and the palette can be represented without a palette. 

868 

869 The current version supports all possible conversions between 

870 "L", "RGB" and "CMYK." The ``matrix`` argument only supports "L" 

871 and "RGB". 

872 

873 When translating a color image to greyscale (mode "L"), 

874 the library uses the ITU-R 601-2 luma transform:: 

875 

876 L = R * 299/1000 + G * 587/1000 + B * 114/1000 

877 

878 The default method of converting a greyscale ("L") or "RGB" 

879 image into a bilevel (mode "1") image uses Floyd-Steinberg 

880 dither to approximate the original image luminosity levels. If 

881 dither is :data:`NONE`, all values larger than 127 are set to 255 (white), 

882 all other values to 0 (black). To use other thresholds, use the 

883 :py:meth:`~PIL.Image.Image.point` method. 

884 

885 When converting from "RGBA" to "P" without a ``matrix`` argument, 

886 this passes the operation to :py:meth:`~PIL.Image.Image.quantize`, 

887 and ``dither`` and ``palette`` are ignored. 

888 

889 :param mode: The requested mode. See: :ref:`concept-modes`. 

890 :param matrix: An optional conversion matrix. If given, this 

891 should be 4- or 12-tuple containing floating point values. 

892 :param dither: Dithering method, used when converting from 

893 mode "RGB" to "P" or from "RGB" or "L" to "1". 

894 Available methods are :data:`NONE` or :data:`FLOYDSTEINBERG` (default). 

895 Note that this is not used when ``matrix`` is supplied. 

896 :param palette: Palette to use when converting from mode "RGB" 

897 to "P". Available palettes are :data:`WEB` or :data:`ADAPTIVE`. 

898 :param colors: Number of colors to use for the :data:`ADAPTIVE` palette. 

899 Defaults to 256.
