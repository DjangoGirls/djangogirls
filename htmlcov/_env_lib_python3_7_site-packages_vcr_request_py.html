102class HeadersDict(CaseInsensitiveDict): 

103 """ 

104 There is a weird quirk in HTTP. You can send the same header twice. For 

105 this reason, headers are represented by a dict, with lists as the values. 

106 However, it appears that HTTPlib is completely incapable of sending the 

107 same header twice. This puts me in a weird position: I want to be able to 

108 accurately represent HTTP headers in cassettes, but I don't want the extra 

109 step of always having to do [0] in the general case, i.e. 

110 request.headers['key'][0] 

111 

112 In addition, some servers sometimes send the same header more than once, 

113 and httplib *can* deal with this situation. 

114 

115 Futhermore, I wanted to keep the request and response cassette format as 

116 similar as possible. 

117 

118 For this reason, in cassettes I keep a dict with lists as keys, but once 

119 deserialized into VCR, I keep them as plain, naked dicts. 

120 """
